Documentation

Everything you need to build blazingly fast web servers with Xeo and Rust.

Installation

Add Xeo to your project via Cargo. No external dependencies required.

terminal 
# Add to existing project
cargo add xeo

# Or create a new project
cargo new my-server
cd my-server
cargo add xeo

Xeo compiles to a single native binary with zero runtime dependencies. No garbage collector, no virtual machine.

Note

Xeo requires Rust 1.75 or later. Run rustc --version to check your version.

Quick Start

Create a minimal server in just a few lines of Rust:

src/main.rs 
use xeo::prelude::*;

#[xeo::main]
async fn main() {
    Xeo::new()
        .route("/", get(|| async { "Hello, World!" }))
        .bind("127.0.0.1:3000")
        .await
        .run()
        .await;
}

Run with cargo run and visit http://localhost:3000 in your browser.

Routing

Xeo provides a type-safe router with compile-time route validation. Define routes using method extractors.

src/main.rs 
use xeo::prelude::*;

async fn get_user(Path(id): Path<u64>) -> String {
    format!("User: {}", id)
}

async fn create_user(Json(body): Json<CreateUser>) -> StatusCode {
    // Handle user creation
    StatusCode::CREATED
}

#[xeo::main]
async fn main() {
    Xeo::new()
        .route("/users/:id", get(get_user))
        .route("/users", post(create_user))
        .bind("0.0.0.0:8080").await
        .run().await;
}

Route parameters are extracted using the Path extractor. Type mismatches are caught at compile time.

  • get(handler) — GET requests
  • post(handler) — POST requests
  • put(handler) — PUT requests
  • delete(handler) — DELETE requests

Extractors

Extractors parse incoming requests into strongly-typed Rust values. They work as handler function arguments.

src/handlers.rs 
use xeo::prelude::*;

#[derive(Deserialize)]
struct QueryParams {
    page: Option<u32>,
    limit: Option<u32>,
}

async fn list_items(
    Query(params): Query<QueryParams>,
    State(db): State<Database>,
) -> Json<Vec<Item>> {
    let items = db.fetch_items(
        params.page.unwrap_or(1),
        params.limit.unwrap_or(20),
    ).await;
    Json(items)
}

State Management

Share application state across handlers using the State extractor. State is thread-safe by default.

src/main.rs 
use xeo::prelude::*;
use std::sync::Arc;

struct AppState {
    db: Database,
    cache: Cache,
}

#[xeo::main]
async fn main() {
    let state = Arc::new(AppState {
        db: Database::connect("postgres://...").await,
        cache: Cache::new(),
    });

    Xeo::new()
        .state(state)
        .route("/", get(handler))
        .bind("0.0.0.0:8080").await
        .run().await;
}

Deployment

Xeo compiles to a single static binary. Deploy anywhere that runs executables — no runtime required.

terminal 
# Build optimized release binary
cargo build --release

# Binary is at target/release/my-server
# Copy and run anywhere
./target/release/my-server

# Or build for specific target
cargo build --release --target x86_64-unknown-linux-musl

Release binaries are typically 2-4MB and start in under 10ms. Perfect for containers and serverless.

Tip

Use cargo build --release for production. Debug builds are significantly slower.
