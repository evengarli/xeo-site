Build with Xeo

The manual for building blazingly fast, file-based web servers using pure Rust and synchronous TCP streams.

Installation

Add Xeo to your Rust project via Cargo.

Cargo.toml 
# Add to dependencies
[dependencies]
xeo = "0.1.5"

Xeo compiles to a single native binary. It relies only on the Rust standard library and the httparse crate for zero-copy parsing.

Quick Start

Create a minimal server in just a few lines of Rust. Xeo uses a builder-like pattern to register files to routes.

src/main.rs 
use xeo::Xeo;

fn main() {
    // 1. Listen on Port 8080
    let mut app = Xeo::new(8080, "0.0.0.0".to_string());

    // 2. Map URL (key) to File (value)
    app.route("/".to_string(), "index.html".to_string());
    app.route("/about".to_string(), "about.html".to_string());

    // 3. Start the loop
    app.server();
}

Run with cargo run and visit http://127.0.0.1:8080 in your browser.

Synchronous Core

Xeo runs on the main thread using standard TCP blocking I/O, ensuring deterministic performance.

Zero-Copy Parsing

Uses httparse to read bytes directly from the stream buffer without unnecessary cloning.

Routing

Xeo uses a unique In-Memory Routing system. Instead of checking the hard drive for every request, Xeo loads your files into RAM at startup.

src/main.rs 
// The first argument is the Browser URL
// The second argument is the File System Path
app.route("/docs".to_string(), "docs.html".to_string());

This decoupling allows you to have "Clean URLs" (like /docs) that point to physical files (docs.html). If a user requests a path that isn't in the HashMap, Xeo returns a 404 immediately.

Architecture

Understanding how Xeo handles a request will help you optimize your deployment.

1. Startup Phase

When you call .route(), Xeo reads the file from disk immediately and stores it in a HashMap.

2. Connection Phase

The .server() loop accepts TCP connections one by one. It reads the byte stream into a 2048-byte buffer.

3. Parsing Phase

The buffer is parsed to extract the Path. Xeo checks the HashMap for an exact match.

4. Response Phase

If found, the content is wrapped in HTTP headers (200 OK) and written back to the stream.

Deployment

Xeo is designed for modern cloud deployments. It compiles to a single binary that can run on any Linux server.

terminal 
# 1. Build for production (Optimized)
cargo build --release

# 2. Run the binary
./target/release/your_app_name
Environment Variables

When deploying to clouds like Render or Heroku, remember to read the PORT environment variable instead of hardcoding 8080.
