Introduction

Xeo is a minimalistic web framework that prioritizes control. Unlike high-level frameworks that hide the details, Xeo exposes the raw TCP stream, giving you complete authority over how bytes are handled.

Installation

To start using Xeo, simply add it to your project dependencies or clone the core files into your library path.

# In your Cargo.toml
[dependencies]
xeo = "0.1.3"

Routing

Xeo uses an In-Memory HashMap to store routes. This means all file reads happen once at startup, ensuring incredible runtime performance.

Tip: The first argument is the browser URL, the second is the file on your disk. They do not have to match! 
use xeo::Xeo;

fn main() {
    // 1. Initialize Server (Port, IP)
    let mut app = Xeo::new(8080, "0.0.0.0".to_string());

    // 2. Register Routes
    app.route("/".to_string(), "index.html".to_string());
    app.route("/docs".to_string(), "docs.html".to_string());

    // 3. Launch
    app.server();
}

Deployment

Ready to go live? Xeo compiles to a single binary. Whether you use a DigitalOcean Droplet or a container, the process is simple.

Always build for release to enable compiler optimizations:

cargo build --release
./target/release/xeo_app
